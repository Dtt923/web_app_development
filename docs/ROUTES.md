# 路由設計文件 (Routes Design)

本文件定義了任務管理系統的 URL 路由規劃、HTTP 方法及對應的處理邏輯與 Jinja2 模板。

## 1. 路由總覽表格

| 功能 | HTTP 方法 | URL 路徑 | 對應模板 | 說明 |
| :--- | :--- | :--- | :--- | :--- |
| 任務列表與新增表單 | GET | / | `index.html` | 顯示首頁，包含所有任務列表與新增任務的表單。 |
| 建立任務 | POST | /tasks | — | 接收新增任務的表單資料，存入資料庫後重導向至 `/`。 |
| 編輯任務頁面 | GET | /tasks/<int:id>/edit | `edit.html` | 顯示特定任務的編輯表單。 |
| 更新任務 | POST | /tasks/<int:id>/update | — | 接收編輯任務的表單資料，更新資料庫後重導向至 `/`。 |
| 刪除任務 | POST | /tasks/<int:id>/delete | — | 將特定任務從資料庫中刪除，並重導向至 `/`。 |
| 切換完成狀態 | POST | /tasks/<int:id>/toggle | — | 改變特定任務的 `is_completed` 狀態，並重導向至 `/`。 |

## 2. 每個路由的詳細說明

### 1. 任務列表與新增表單
- **URL**: `/`
- **HTTP 方法**: `GET`
- **處理邏輯**: 呼叫 Model 取得資料庫中所有的任務列表。可以透過 URL 參數進行過濾（例如 `?status=completed` 或 `?status=pending`）。
- **輸出**: 渲染 `index.html` 並傳遞 `tasks` 變數。
- **錯誤處理**: 無特殊錯誤處理，若無任務則顯示空列表。

### 2. 建立任務
- **URL**: `/tasks`
- **HTTP 方法**: `POST`
- **輸入**: 表單欄位包含 `title` (必填)、`description`、`due_date`。
- **處理邏輯**:
  - 驗證 `title` 是否為空。
  - 呼叫 Model 將資料插入 `tasks` 表格。
- **輸出**: 成功後重導向至 `/`。
- **錯誤處理**: 若資料驗證失敗（例如無標題），可重導向回 `/` 並帶有錯誤訊息，或渲染 `index.html` 顯示錯誤。

### 3. 編輯任務頁面
- **URL**: `/tasks/<int:id>/edit`
- **HTTP 方法**: `GET`
- **處理邏輯**: 根據 `id` 呼叫 Model 查詢對應的任務。
- **輸出**: 若任務存在，渲染 `edit.html` 並傳遞該 `task` 變數。
- **錯誤處理**: 若該 `id` 的任務不存在，則回傳 404 Not Found 或重導向回 `/` 並顯示錯誤。

### 4. 更新任務
- **URL**: `/tasks/<int:id>/update`
- **HTTP 方法**: `POST`
- **輸入**: 表單欄位包含 `title` (必填)、`description`、`due_date`。
- **處理邏輯**:
  - 驗證該任務是否存在。
  - 驗證 `title` 是否為空。
  - 呼叫 Model 更新 `tasks` 表格中對應 `id` 的紀錄。
- **輸出**: 成功後重導向至 `/`。
- **錯誤處理**: 若資料驗證失敗，重導向回編輯頁面並顯示錯誤。若找不到任務，回傳 404。

### 5. 刪除任務
- **URL**: `/tasks/<int:id>/delete`
- **HTTP 方法**: `POST`
- **處理邏輯**: 根據 `id` 呼叫 Model 刪除該任務。
- **輸出**: 成功後重導向至 `/`。
- **錯誤處理**: 若找不到任務，回傳 404 或直接重導向至 `/`。

### 6. 切換完成狀態
- **URL**: `/tasks/<int:id>/toggle`
- **HTTP 方法**: `POST`
- **處理邏輯**: 根據 `id` 尋找任務，將其 `is_completed` 值反轉 (0 變 1，1 變 0)，並更新 `updated_at`。
- **輸出**: 重導向至 `/`。
- **錯誤處理**: 若找不到任務，回傳 404 或直接重導向至 `/`。

## 3. Jinja2 模板清單

- **`base.html`**: 網站的共用骨架，包含 HTML `<head>` 標籤、CSS 引用、共用的導覽列與 Footer。其他模板皆繼承此檔案。
- **`index.html`**: 繼承自 `base.html`。首頁視圖，包含「新增任務表單」與「任務列表」展示。任務列表會疊代傳入的 `tasks` 變數來渲染每個任務項目。
- **`edit.html`**: 繼承自 `base.html`。單一任務編輯頁面，包含一個預填好該任務資料的表單，讓使用者修改後送出。
