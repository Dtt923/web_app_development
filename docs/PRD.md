# 任務管理系統 PRD (Product Requirements Document)

## 1. 專案概述

### 背景與動機
在現代快節奏的工作與生活中，個人與團隊面臨著繁雜的待辦事項與時間壓力。我們需要一個直覺、高效的系統來集中管理日常任務，避免遺漏重要事項，並提升整體生產力與時間管理能力。

### 目標用戶
- **職場上班族**：需要管理跨專案、多重任務的工作者。
- **學生**：需要追蹤作業、考試進度的人。
- **一般個人**：希望隨時記錄生活雜事和個人計畫的使用者。

### 核心價值主張
提供一個簡潔、易用的平台，讓使用者能快速建立及管理每日任務，掌握進度與歷史紀錄，打造清爽高效的個人時間管理體驗。

## 2. 功能需求

以下為系統核心功能，整理為 User Story 格式：

### 1. 新增任務
- **作為** 一般使用者，**我希望** 能快速新增一個任務（包含標題、描述、到期日等資訊），**以便** 我能清楚記錄待辦事項。

### 2. 編輯任務
- **作為** 一般使用者，**我希望** 能夠隨時修改已建立任務的內容細節，**以便** 在計畫變更時保持任務資訊的正確性。

### 3. 刪除任務
- **作為** 一般使用者，**我希望** 能夠刪除不再需要或誤建的任務，**以便** 保持任務清單的清楚整齊。

### 4. 標記完成
- **作為** 一般使用者，**我希望** 能在任務執行完畢後將其標記為「已完成」，**以便** 我清楚知道哪些事情已經結案，並獲得成就感。

### 5. 任務紀錄
- **作為** 一般使用者，**我希望** 能查看任務的歷史紀錄（包含建立時間、完成時間等），**以便** 我日後回顧我的執行進度與歷史工作表現。

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Flask 進行開發。
- **前端模板**：使用 Jinja2 渲染 HTML 視圖。
- **資料庫**：採用 SQLite 作為輕量級的資料儲存方案。

### 效能與安全考量
- **響應速度**：確保頁面加載及所有任務操作能快速響應。
- **安全性**：防範 SQL Injection 與 XSS 等常見網路安全問題，確保系統資料存取安全。
- **使用者體驗**：需要有基本的錯誤處理與友善的操作提示。

## 4. MVP 範圍

### Must Have (必須擁有)
- 任務的 CRUD (建立、讀取列表、更新/編輯、刪除)。
- 將任務在未完成與完成狀態之間切換。
- 基礎的介面排版佈局。

### Should Have (應該擁有)
- 任務狀態與時間的紀錄顯示（如建立時間、最後更新時間）。
- 能夠依據狀態（全部、已完成、未完成）過濾並排序清單。

### Nice to Have (加分項目)
- 任務類別或標籤功能。
- 多使用者登入與註冊（用戶個人化資料）。
- 個別任務的優先級設定（高、中、低）。
